## quando erro ao reiniciar o container
| Comando | O que faz |
|---------|-----------|
| `docker compose --profile tickmill stop mt5-tickmill` | Para o container Tickmill com SIGTERM (MT5, VNC, bridge encerram de forma ordenada). |
| `docker compose --profile tickmill rm -f mt5-tickmill` | Remove o container parado. O volume `mt5_tickmill_data` (Wine/MT5) **permanece**; só some o filesystem efêmero, inclusive `/tmp/.X99-lock`. |
| `docker compose --profile tickmill up -d mt5-tickmill` | Cria um container **novo** e sobe em background (`-d`), rodando o entrypoint do zero (VNC, MT5, bridge). |

**Resumo:** para → apaga o container problemático → sobe outro limpo, mantendo login, terminal e config no volume.
